# WeaveNet - Архитектура проекта

## Обзор

WeaveNet - это стратегическая игра с гексагональной картой, где игроки строят сооружения и управляют ресурсами. Проект построен на модульной архитектуре с чётким разделением ответственности между компонентами.

## Структура проекта

```
WeaveNet/
├── index.html          # Главный HTML-файл с разметкой и стилями
├── README.md           # Документация проекта (этот файл)
└── src/
    └── modules/
        ├── game.js         # Главный класс игры
        ├── hexGrid.js      # Гексагональная сетка
        ├── gridRenderer.js # Отрисовка графики
        ├── buildings.js    # Типы построек
        └── userManager.js  # Управление пользователями
```

## Модули

### 1. `index.html`

**Назначение:** Главный HTML-файл, содержащий разметку интерфейса, стили и подключение скриптов.

**Основные компоненты:**
- **Верхняя панель (`#top-bar`)**: Отображение ресурсов (Информация, Энергия), таймер игры, кнопки скорости
- **Игровое поле (`#game-canvas`)**: Canvas элемент для отрисовки гексагональной карты
- **Нижняя панель (`#bottom-panel`)**: Информация о выбранном гексе (координаты, тип местности, постройка, пользователи, соседи)
- **Панель построек**: Кнопки выбора типа постройки для строительства

**Стили:** Встроенные CSS стили для тёмной темы, tooltip подсказок, адаптивного интерфейса.

---

### 2. `game.js` - Главный класс игры

**Назначение:** Централизованное управление игрой, объединение всех модулей, обработка ввода, игровой цикл.

**Основные обязанности:**
- Инициализация всех игровых систем (HexGrid, GridRenderer, UserManager, Buildings)
- Обработка событий мыши (клики, движение)
- Управление режимом строительства (выбор постройки, размещение)
- Обновление ресурсов игрока
- Запуск и поддержка игрового цикла (update + render)

**Ключевые методы:**
- `constructor()` - Создание и настройка всех компонентов
- `start()` - Запуск игрового цикла
- `update()` - Обновление логики игры (перемещение пользователей)
- `render()` - Отрисовка кадра
- `handleCanvasClick()` - Обработка клика по полю (выбор гекса, строительство)
- `handleMouseMove()` - Подсветка гекса под курсором, обновление UI
- `selectBuildingType()` - Выбор типа постройки для строительства
- `placeBuilding()` - Размещение постройки на гексе
- `demolishBuilding()` - Снос постройки с возвратом ресурсов
- `updateBottomPanel()` - Обновление информации в нижней панели
- `updateBottomPanelUI()` - Частичное обновление UI (счётчик пользователей)

**Взаимодействия:**
- Создаёт экземпляры всех модулей
- Передаёт события от пользователя соответствующим модулям
- Вызывает рендерер для отрисовки каждого кадра

---

### 3. `hexGrid.js` - Гексагональная сетка

**Назначение:** Создание и управление гексагональной картой, хранение состояния гексов.

**Основные обязанности:**
- Генерация гексагональной сетки (концентрические кольца)
- Хранение данных о каждом гексе (ID, координаты, тип местности, постройка, связи)
- Управление связями между соседними гексами
- Размещение и удаление построек на гексах

**Структура гекса:**
```javascript
{
    id: number,           // Уникальный идентификатор
    q: number,            // Гексагональная координата q
    r: number,            // Гексагональная координата r
    terrain: string,      // Тип местности ('plains', 'forest', 'mountain', 'water')
    building: Object|null,// Объект постройки или null
    neighbors: Object     // Соседние гексы { '0': hex, '1': hex, ... }
}
```

**Ключевые методы:**
- `constructor(radius)` - Создание сетки с указанным радиусом
- `setupNeighbors()` - Автоматическая установка связей между соседями
- `getHexById(id)` - Получение гекса по ID
- `getAllHexes()` - Получение всех гексов
- `placeBuilding(hexId, building)` - Размещение постройки
- `removeBuilding(hexId)` - Удаление постройки

**Алгоритм генерации:**
1. Создаётся центральный гекс (0, 0)
2. Для каждого кольца от 1 до radius:
   - Проход по 6 направлениям
   - Создание гексов вдоль каждой стороны кольца

---

### 4. `gridRenderer.js` - Отрисовка графики

**Назначение:** Визуализация игрового поля, гексов, построек, пользователей и эффектов.

**Основные обязанности:**
- Отрисовка гексагональной сетки
- Отрисовка построек на гексах
- Отрисовка пользователей (кружочки на гексах)
- Эффекты выделения (свечение выбранного гекса)
- Подсветка при наведении (hover)
- Отрисовка "призрака" постройки при строительстве

**Ключевые методы:**
- `constructor(canvas, hexGrid)` - Инициализация рендерера
- `render()` - Отрисовка всего поля
- `drawHex(x, y, color, options)` - Отрисовка одного гекса
- `drawBuilding(hex, x, y)` - Отрисовка постройки на гексе
- `drawUsers(users, x, y)` - Отрисовка пользователей
- `drawGhostBuilding(hex, x, y, buildingType)` - Отрисовка призрака постройки
- `getHexAtPosition(mouseX, mouseY)` - Определение гекса под курсором
- `setSelectedHex(hexId)` - Установка выбранного гекса
- `setHoveredHex(hexId)` - Установка гекса под курсором
- `setBuildingTypeToPlace(type)` - Установка типа постройки для preview

**Особенности рендеринга:**
- Использует Canvas API для производительности
- Применяет эффекты свечения через `shadowBlur`
- Отрисовка в слоях: фон → гексы → постройки → пользователи → эффекты

---

### 5. `buildings.js` - Типы построек

**Назначение:** Определение типов зданий, их характеристик и логика создания.

**Основные обязанности:**
- Хранение данных о типах построек (название, стоимость, описание, цвет)
- Создание объектов построек
- Проверка возможности строительства на гексе
- Механика улучшения построек

**Типы построек:**
1. **Вышка (tower)** - Базовая постройка для покрытия сигналом
2. **Ретранслятор (repeater)** - Усиливает сигнал, увеличивает радиус
3. **Генератор (generator)** - Производит энергию для других построек

**Структура типа постройки:**
```javascript
{
    id: string,           // Уникальный идентификатор
    name: string,         // Название для отображения
    cost: number,         // Стоимость в информации
    description: string,  // Описание для tooltip
    color: string,        // Цвет для отрисовки
    production: {         // Производство ресурсов
        info: number,
        energy: number
    },
    range: number,        // Радиус действия
    level: number         // Уровень постройки
}
```

**Ключевые методы:**
- `constructor()` - Инициализация списка типов построек
- `getAllBuildingTypes()` - Получение всех типов
- `getBuildingTypeById(id)` - Получение типа по ID
- `createBuilding(typeId)` - Создание объекта постройки
- `canBuildOnHex(hex, buildingType)` - Проверка возможности строительства
- `upgradeBuilding(building)` - Улучшение постройки

---

### 6. `userManager.js` - Управление пользователями

**Назначение:** Создание, хранение и обновление пользователей, перемещающихся по карте.

**Основные обязанности:**
- Создание новых пользователей со случайными позициями
- Перемещение пользователей между гексами (раз в секунду)
- Ограничение количества пользователей (макс 1.5 × число гексов)
- Ограничение заполненности гекса (макс 6 пользователей на гекс)

**Структура пользователя:**
```javascript
{
    id: number,           // Уникальный идентификатор
    hexId: number,        // ID текущего гекса
    createdAt: number,    // Время создания
    color: string,        // Цвет для отрисовки
    moveCooldown: number  // Задержка перед следующим перемещением
}
```

**Логика перемещения:**
1. Каждую секунду каждый пользователь имеет 50% шанс переместиться
2. Выбирается случайный соседний гекс
3. Перемещение возможно только если на целевом гексе < 6 пользователей

**Ключевые методы:**
- `constructor(hexGrid)` - Инициализация менеджера
- `createUser()` - Создание нового пользователя
- `update()` - Обновление всех пользователей (перемещение)
- `updateUser(user)` - Обновление одного пользователя
- `getUsersOnHex(hexId)` - Получение пользователей на гексе
- `getUserCountOnHex(hexId)` - Количество пользователей на гексе
- `populateInitialUsers(count)` - Заполнение карты начальными пользователями

## Поток данных

1. **Инициализация:**
   ```
   index.html → Game.constructor() → создание всех модулей
   ```

2. **Игровой цикл:**
   ```
   Game.start() → requestAnimationFrame → Game.update() → Game.render()
   ```

3. **Обработка ввода:**
   ```
   Событие мыши → Game.handleCanvasClick/MouseMove → 
   → HexGrid/GridRenderer → обновление состояния → render()
   ```

4. **Перемещение пользователей:**
   ```
   Game.update() → UserManager.update() → перемещение → 
   → updateBottomPanelUI() → обновление счётчика
   ```

## Взаимодействие модулей

```
                    Game.js (Controller)
                   /        |        \
                  /         |         \
           HexGrid    GridRenderer   UserManager
              |            |             |
              |            |             |
           Buildings -------+-------------+
```

- **Game** - центральный контроллер, координирует все модули
- **HexGrid** - модель данных карты
- **GridRenderer** - представление (view)
- **UserManager** - модель пользователей
- **Buildings** - фабрика и данные о постройках

## Расширение

Для добавления нового функционала:

1. **Новый тип постройки:** Добавить в `buildings.js` в конструктор
2. **Новый тип местности:** Изменить генерацию в `hexGrid.js`
3. **Новое действие:** Добавить обработчик в `game.js`
4. **Визуальный эффект:** Добавить метод отрисовки в `gridRenderer.js`
