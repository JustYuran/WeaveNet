# 📘 Документация модульной архитектуры WeaveNet

## 📁 Структура проекта

```
WeaveNet/
├── index.html              # Главный HTML файл с подключением модулей
├── src/
│   └── modules/
│       ├── hexGrid.js      # Модуль гексагональной сетки и карты
│       ├── gridRenderer.js # Модуль отрисовки графики
│       ├── economyManager.js # Модуль экономики и ресурсов
│       ├── saveManager.js  # Модуль сохранений и загрузки
│       └── game.js         # Основной класс игры (интеграция)
```

## 🔧 Описание модулей

### 1. HexGrid (`hexGrid.js`)
**[ЧТО]** Базовый класс для работы с гексагональной координатной системой и картой мира.

**[ЗАЧЕМ]** Инкапсулирует логику:
- Конвертации экранных координат в гексагональные и обратно
- Генерации карты формы России с узнаваемыми географическими элементами
- Управления размещением объектов на гексах

**[PLAN]** 1.1. Гексагональная основа, 1.4. Схожесть карты на реальную карту

**Основные методы:**
- `screenToHex(x, y)` - конвертация экрана в гекс
- `hexToScreen(q, r)` - конвертация гекса в экран
- `initMap(width, height)` - генерация карты
- `canPlace(q, r)` - проверка возможности размещения
- `placeObject(q, r, obj)` - размещение объекта
- `removeObject(q, r)` - удаление объекта

---

### 2. GridRenderer (`gridRenderer.js`)
**[ЧТО]** Класс для рендеринга карты, гексов, объектов и зон покрытия на canvas.

**[ЗАЧЕМ]** Инкапсулирует всю графику игры:
- Послойную отрисовку (гексы → зоны → объекты)
- Плавную интерполяцию цветов при смене режима
- Объединение пересекающихся зон покрытия

**[PLAN]** 2.3. Визуализация покрытия, 6.3. Навигация по карте

**Основные методы:**
- `draw(ctx)` - основная отрисовка всей сцены
- `drawHex(ctx, x, y, size, color, obstacle)` - рисование гекса
- `drawObject(ctx, x, y, size, object)` - рисование объекта
- `drawCoverageMerged(ctx, mode)` - отрисовка объединённых зон
- `invalidateCoverageCache()` - сброс кэша зон

---

### 3. EconomyManager (`economyManager.js`)
**[ЧТО]** Класс для расчёта производства/потребления ресурсов.

**[ЗАЧЕМ]** Управляет экономической моделью:
- Генерацией информации активными пользователями
- Потреблением энергии зданиями
- Балансом потоков ресурсов

**[PLAN]** 4. Экономика

**Основные методы:**
- `update(deltaTime, speed)` - расчёт изменений за кадр
- `canAfford(cost)` - проверка достаточности ресурсов
- `spendInfo(cost)` - списание информации
- `refundInfo(cost)` - возврат при сносе (50%)
- `getState()` - получение текущего состояния

---

### 4. SaveManager (`saveManager.js`)
**[ЧТО]** Класс для автосохранения, загрузки и экспорта конфигурации.

**[ЗАЧЕМ]** Обеспечивает сохранность прогресса:
- Автоматическое сохранение каждые 5 минут
- Ручные слоты сохранений
- Экспорт/импорт конфигураций в JSON

**[PLAN]** 8.3. Сохранение и экспорт

**Основные методы:**
- `autoSave()` - автоматическое сохранение
- `manualSave(slotName)` - ручное сохранение
- `loadLastSave()` - загрузка последнего сохранения
- `exportConfig()` - экспорт в JSON строку
- `importConfig(jsonString)` - импорт из JSON
- `resetProgress()` - полный сброс прогресса

---

### 5. Game (`game.js`)
**[ЧТО]** Центральный класс игры, объединяющий все системы.

**[ЗАЧЕМ]** Координирует работу модулей:
- Создаёт экземпляры всех менеджеров
- Обрабатывает ввод пользователя (мышь, зум)
- Управляет игровым циклом (update + render)
- Связывает UI с игровой логикой

**[PLAN]** Все разделы (интеграция)

**Основные методы:**
- `init()` - начальная настройка игры
- `setupEventListeners()` - обработчики событий
- `selectHex(q, r)` - выбор гекса
- `buildRouter(key)` / `buildTower(key)` - строительство
- `toggleMode(key)` - смена режима
- `demolish(key)` - снос здания
- `loop()` - главный игровой цикл

---

## 🔄 Порядок подключения модулей

Модули должны быть подключены в строгом порядке зависимостей:

```html
<script src="src/modules/hexGrid.js"></script>
<script src="src/modules/gridRenderer.js"></script>
<script src="src/modules/economyManager.js"></script>
<script src="src/modules/saveManager.js"></script>
<script src="src/modules/game.js"></script>
```

**Зависимости:**
1. `hexGrid.js` - не зависит от других модулей
2. `gridRenderer.js` - требует `HexGrid`
3. `economyManager.js` - требует `HexGrid`
4. `saveManager.js` - требует `Game`
5. `game.js` - требует все предыдущие модули

## 📝 Формат комментариев

Все комментарии следуют формату **[ЧТО] → [ЗАЧЕМ] → [ССЫЛКА НА ПЛАН]**:

```javascript
/**
 * [ЧТО] Проверка возможности размещения объекта на гексе.
 * [ЗАЧЕМ] Предотвращает размещение на преградах или занятых гексах.
 * [PLAN] 1.3. Логика размещения объектов
 */
canPlace(q, r) { ... }
```

## 🚀 Расширение архитектуры

Для добавления нового модуля:

1. Создайте файл в `/src/modules/yourModule.js`
2. Добавьте JSDoc комментарий с описанием
3. Следуйте формату комментариев [ЧТО] → [ЗАЧЕМ] → [PLAN]
4. Подключите в `index.html` в правильном порядке
5. Импортируйте в `game.js` если требуется интеграция
