Все файлы находятся в корне /workspace.

⚙️ АЛГОРИТМ РАБОТЫ (СТРОГО СОБЛЮДАЙ ПОРЯДОК):

ШАГ 1: АНАЛИЗ АРХИТЕКТУРЫ (ОБЯЗАТЕЛЬНО)
- В самую первую очередь открой и изучи /workspace/MODULES_README.md.
- Пойми структуру проекта, связи между файлами и принятые паттерны.
- Весь новый код должен строго соответствовать описанной архитектуре.

ШАГ 2: ПРОВЕРКА ОШИБОК
- Открой папку /workspace/ERROR.
- Если там есть отчёты о багах — ТВОЙ АБСОЛЮТНЫЙ ПРИОРИТЕТ их исправить.
- Если папка пуста или ошибки исправлены — переходи к ШАГУ 3.

ШАГ 3: АНАЛИЗ ПЛАНА И ФОРМИРОВАНИЕ TODO
- Открой /workspace/Rules/plan.txt.
- Найди все пункты, которые ещё не реализованы, не работают или отмечены знаком "-" / "в ожидании".
- На основе этих пунктов сформируй /workspace/Rules/TODO.md.
- ⚠️ ВАЖНО: Выполняй задачи СТРОГО ПО ПОРЯДКУ ВОЗРАСТАНИЯ НОМЕРОВ (1.1 → 1.2 → 1.3 → 2.1 и т.д.). Не перепрыгивай через пункты.

ШАГ 4: КОДИРОВАНИЕ И ОТЧЕТНОСТЬ
- Реализуй код для текущего пункта из TODO.
- После завершения обнови /workspace/Rules/plan.txt: отметь выполненные пункты как готовые.
- Обнови /workspace/Rules/TODO.md: убери сделанное, оставь только предстоящие задачи.
- Создай новый файл в /workspace/ChangeLog/:
  • Имя: "ChangeLog v{версия}.txt"
  • Версионирование: +1 к последнему числу. Если 9 → сброс в 0 и +1 к предпоследнему (0.0.0.9 → 0.0.1.0).
  • Содержание: что изменено, какие файлы затронуты.
- Если во время работы возникли критические блокирующие баги — создай отчёт в /workspace/ERROR/.

📝 ПРАВИЛО КОММЕНТИРОВАНИЯ КОДА (ОБЯЗАТЕЛЬНО):
Каждый файл, класс, функция и сложный блок кода должны содержать подробные комментарии на русском языке.
Формат комментария: [ЧТО] → [ЗАЧЕМ] → [ССЫЛКА НА ПЛАН]

Пример оформления:
// [ЧТО] Функция расчёта стабильности сети на основе избыточных связей.
// [ЗАЧЕМ] Позволяет игроку видеть, насколько сеть устойчива к обрывам. Узлы с <2 связями считаются уязвимыми, что влияет на геймплей при климатических событиях.
// [PLAN] 7.3.1 Избыточность путей
function calculateStability(nodes: NetworkNode[]): number { ... }

Без таких комментариев код не считается принятым.
