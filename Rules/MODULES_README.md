# 📚 Архитектура модулей WeaveNet

## Обзор архитектуры

WeaveNet — браузерная стратегия с модульной архитектурой, где каждый модуль отвечает за определённый аспект игры. Все модули расположены в `/WeaveNet/src/modules/`.

```
WeaveNet/src/modules/
├── game.js          # Главный класс игры (оркестратор)
├── hexGrid.js       # Логика гексагональной сетки
└── gridRenderer.js  # Отрисовка графики на canvas
```

---

## 🔷 Модули

### 1. `game.js` — Главный класс игры

**[ЧТО]** Объединяет все модули и управляет игровым процессом  
**[ЗАЧЕМ]** Централизованное управление игрой, обработка ввода, игровой цикл

#### Основные компоненты:

```javascript
class Game {
    constructor() {
        this.canvas           // HTML5 canvas для отрисовки
        this.hexGrid          // Экземпляр HexGrid (логика сетки)
        this.gridRenderer     // Экземпляр GridRenderer (отрисовка)
        this.buildingTypes    // Доступные типы построек
        this.resources        // Ресурсы игрока (info, energy)
        this.selectedBuildingType  // Выбранный тип постройки
        this.animationId      // ID requestAnimationFrame
    }
}
```

#### Ключевые методы:

| Метод | Описание |
|-------|----------|
| `constructor()` | Инициализация всех систем игры |
| `createBuildingPanel()` | Создание UI панели построек |
| `setupEventListeners()` | Настройка обработчиков мыши |
| `updateResourceUI()` | Обновление отображения ресурсов |
| `start()` | Запуск игрового цикла |
| `gameLoop()` | Основной цикл (обновление + отрисовка) |
| `handleCanvasClick()` | Обработка кликов по полю |
| `placeBuilding(hex)` | Размещение постройки на гексе |

#### Взаимодействие с другими модулями:
- Создаёт `HexGrid(60)` — сетку из 7 гексов
- Создаёт `GridRenderer(canvas, hexGrid)` — рендерер
- Обрабатывает ввод пользователя и передаёт команды в другие модули

### 2. `hexGrid.js` — Гексагональная сетка

**[ЧТО]** Создаёт статическое поле из 7 гексов (1 центр + 6 вокруг) в форме круга  
**[ЗАЧЕМ]** Упрощённая карта для тестирования механик без проблем с генерацией

#### Структура данных гекса:

```javascript
{
    id: number,           // Уникальный идентификатор
    q: number,            // Axial-координата q
    r: number,            // Axial-координата r
    neighbors: [6],       // Ссылки на соседние гексы
    building: null,       // Постройка на гексе (если есть)
    x: number,            // Pixel-координата X (для отрисовки)
    y: number             // Pixel-координата Y (для отрисовки)
}
```

#### Ключевые методы:

| Метод | Описание |
|-------|----------|
| `constructor(hexSize)` | Инициализация сетки с заданным размером гекса |
| `generateHexGrid()` | Генерация 7 гексов (центр + 6 соседей) |
| `createHex(q, r, id)` | Создание объекта гекса с axial-координатами |
| `getOppositeDirection(dir)` | Получение противоположного направления (0↔3, 1↔4, 2↔5) |
| `getAllHexes()` | Возврат массива всех гексов |
| `getHexSize()` | Возврат размера гекса |
| `getHexWidth()` | Возврат ширины гекса (`sqrt(3) * size`) |
| `getHexHeight()` | Возврат высоты гекса (`2 * size`) |

#### Axial-координаты:

Для flat-top ориентации (плоская вершина):
- **Направления:** 0=E, 1=NE, 2=NW, 3=W, 4=SW, 5=SE
- **Соседи от центра (0,0):**
  - dq = [1, 0, -1, -1, 0, 1]
  - dr = [0, 1, 1, 0, -1, -1]

#### Конвертация координат:

```javascript
// Axial → Pixel (относительно центра экрана)
x = centerX + hexWidth * q
y = centerY + hexHeight * (r + q / 2)
```

### 3. `gridRenderer.js` — Отрисовка графики

**[ЧТО]** Рендерит 7 гексов, постройки и эффекты выделения  
**[ЗАЧЕМ]** Визуализация игрового поля для взаимодействия с игроком

#### Основные компоненты:

```javascript
class GridRenderer {
    constructor(canvas, hexGrid) {
        this.canvas              // HTML5 canvas элемент
        this.ctx                 // 2D контекст для рисования
        this.hexGrid             // Ссылка на HexGrid
        this.selectedHexId       // ID выделенного гекса
        this.buildingTypeToPlace // Тип постройки для размещения
    }
}
```

#### Ключевые методы:

| Метод | Описание |
|-------|----------|
| `constructor(canvas, hexGrid)` | Инициализация рендерера |
| `resizeCanvas()` | Подстройка canvas под размер окна |
| `render()` | Основная функция отрисовки всего кадра |
| `drawHex(hex)` | Отрисовка одного гекса (шестиугольник) |
| `drawBuilding(hex)` | Отрисовка постройки на гексе |
| `drawBuildCursor()` | Отрисовка курсора строительства |
| `setSelectedHex(hexId)` | Установка выделенного гекса |
| `setBuildingTypeToPlace(type)` | Установка типа постройки для размещения |

#### Процесс отрисовки:

1. Очистка canvas (`clearRect`)
2. Вычисление центра экрана
3. Для каждого гекса:
   - Конвертация axial → pixel координат
   - Отрисовка шестиугольника
   - Отрисовка постройки (если есть)
4. Отрисовка курсора строительства (если активен)

#### Отрисовка гекса:

```javascript
// 6 вершин шестиугольника (flat-top)
for (let i = 0; i < 6; i++) {
    const angle = (i * 60) * (Math.PI / 180);
    const x = hex.x + size * Math.cos(angle);
    const y = hex.y + size * Math.sin(angle);
    // ... добавление вершин в путь
}
```

## 🎮 Игровой цикл

```javascript
// В game.js
gameLoop() {
    this.gridRenderer.render();  // Отрисовка
    requestAnimationFrame(() => this.gameLoop());  // Следующий кадр
}
```

**Частота обновления:** ~60 FPS (зависит от монитора)

## 📝 Комментарии в коде

Все модули используют единую систему комментариев:

- **[ЧТО]** — Что делает код
- **[ЗАЧЕМ]** — Зачем это нужно
- **[PLAN]** — Планы на улучшение
- **[ERROR]** — Исправленные ошибки

Пример:
```javascript
// [ЧТО] Создаём центральный гекс с координатами (0, 0)
// [ЗАЧЕМ] Это точка отсчёта для всей сетки
// [PLAN] Добавить случайный тип местности
const centerHex = this.createHex(0, 0, hexId++);
```
