# 📚 Архитектура модулей WeaveNet

## Обзор архитектуры

WeaveNet — браузерная стратегия с модульной архитектурой, где каждый модуль отвечает за определённый аспект игры. Все модули расположены в `/WeaveNet/src/modules/`.

```
WeaveNet/src/modules/
├── game.js          # 🟢 Главный оркестратор: цикл, ввод, ресурсы, связь модулей
├── hexGrid.js       # 🟢 Логика сетки: axial-координаты, соседи, карта 200×200 со случайной генерацией
├── gridRenderer.js  # 🟢 Рендеринг: Canvas отрисовка гексов, объектов, курсоров, эффектов
├── camera.js        # 🟢 Камера: панорамирование (ЛКМ), зум (колесо), ограничения
├── buildings.js     # 🟢 Типы построек: характеристики, стоимость, производство
└── userManager.js   # 🟢 Пользователи: спавн, перемещение, лимиты (до 6 на гекс)
```

## 🔷 Модули

### 1. `game.js` — Главный класс игры (Оркестратор)

**[ЧТО]** Объединяет все модули и управляет игровым процессом  
**[ЗАЧЕМ]** Централизованное управление игрой, обработка ввода, игровой цикл

#### Основные компоненты:

```javascript
class Game {
    constructor() {
        this.canvas              // HTML5 canvas для отрисовки
        this.hexGrid             // Экземпляр HexGrid (сетка 200×200)
        this.cameraManager       // Экземпляр CameraManager (камера)
        this.gridRenderer        // Экземпляр GridRenderer (отрисовка)
        this.userManager         // Экземпляр UserManager (пользователи)
        this.buildingsManager    // Экземпляр Buildings (постройки)
        this.resources           // Ресурсы игрока (info, energy)
        this.selectedBuildingType // Выбранный тип постройки
        this.animationId         // ID requestAnimationFrame
    }
}
```

#### Ключевые методы:

| Метод | Описание |
|-------|----------|
| `constructor()` | Инициализация всех систем игры |
| `createBuildingPanel()` | Создание UI панели построек сверху экрана |
| `setupEventListeners()` | Настройка обработчиков мыши |
| `updateResourceUI()` | Обновление отображения ресурсов |
| `start()` | Запуск игрового цикла |
| `gameLoop()` | Основной цикл (обновление + отрисовка) |
| `handleCanvasClick()` | Обработка кликов по полю |
| `placeBuilding(hex)` | Размещение постройки на гексе |

#### Взаимодействие с другими модулями:
- Создаёт `HexGrid(60, 200)` — сетку 200×200 со случайной генерацией формы
- Создаёт `CameraManager(canvas, hexGrid)` — камеру
- Создаёт `GridRenderer(canvas, hexGrid, cameraManager)` — рендерер
- Создаёт `UserManager(hexGrid)` — менеджер пользователей
- Создаёт `Buildings()` — менеджер типов построек
- Обрабатывает ввод пользователя и передаёт команды в другие модули

### 2. `hexGrid.js` — Гексагональная сетка

**[ЧТО]** Создаёт поле размером 200×200 гексов со случайно генерируемой формой континента  
**[ЗАЧЕМ]** Большая карта для масштабного геймплея с уникальной конфигурацией каждый запуск

#### Структура данных гекса:

```javascript
{
    id: number,           // Уникальный идентификатор
    q: number,            // Axial-координата q
    r: number,            // Axial-координата r
    neighbors: [6],       // Ссылки на соседние гексы
    building: null,       // Постройка на гексе (если есть)
    x: number,            // Pixel-координата X (для отрисовки)
    y: number             // Pixel-координата Y (для отрисовки)
}
```

#### Ключевые методы:

| Метод | Описание |
|-------|----------|
| `constructor(hexSize, mapSize)` | Инициализация сетки 200×200 |
| `generateHexGrid()` | Генерация гексов со случайной формой континента |
| `createHex(q, r, id)` | Создание объекта гекса с axial-координатами и биомом |
| `generateContinentShape(q, r, halfSize)` | Проверка попадания в процедурно сгенерированную форму |
| `getTerrainForHex(q, r)` | Определение биома с плавными переходами (шум + проверка соседей) |
| `getNeighborTerrains(q, r, currentTerrain)` | Проверка соседей для предотвращения изолированных гексов |
| `setupNeighbors()` | Установка связей между соседями |
| `getAllHexes()` | Возврат массива всех гексов |
| `getHexSize()` | Возврат размера гекса |
| `getHexWidth()` | Возврат ширины гекса (`sqrt(3) * size`) |
| `getHexHeight()` | Возврат высоты гекса (`2 * size`) |

#### Axial-координаты:

Для flat-top ориентации (плоская вершина):
- **Направления:** 0=E, 1=NE, 2=NW, 3=W, 4=SW, 5=SE
- **Соседи от центра (0,0):**
  - dq = [1, 0, -1, -1, 0, 1]
  - dr = [0, 1, 1, 0, -1, -1]

#### Конвертация координат:

```javascript
// Axial → Pixel (относительно центра экрана)
x = centerX + hexWidth * q
y = centerY + hexHeight * (r + q / 2)
```

### 3. `gridRenderer.js` — Отрисовка графики

**[ЧТО]** Рендерит гексы, постройки, пользователей и эффекты выделения  
**[ЗАЧЕМ]** Визуализация игрового поля для взаимодействия с игроком

#### Основные компоненты:

```javascript
class GridRenderer {
    constructor(canvas, hexGrid, cameraManager) {
        this.canvas              // HTML5 canvas элемент
        this.ctx                 // 2D контекст для рисования
        this.hexGrid             // Ссылка на HexGrid
        this.cameraManager       // Ссылка на CameraManager
        this.userManager         // Ссылка на UserManager (для отрисовки пользователей)
        this.selectedHexId       // ID выделенного гекса
        this.hoveredHexId        // ID гекса под курсором
        this.buildingTypeToPlace // Тип постройки для размещения
    }
}
```

#### Ключевые методы:

| Метод | Описание |
|-------|----------|
| `constructor(canvas, hexGrid, cameraManager)` | Инициализация рендерера |
| `resizeCanvas()` | Подстройка canvas под размер окна |
| `render()` | Основная функция отрисовки всего кадра |
| `drawHex(hex)` | Отрисовка одного гекса (шестиугольник) |
| `drawBuilding(hex)` | Отрисовка постройки на гексе |
| `drawUsers()` | Отрисовка пользователей на гексах |
| `drawBuildCursor()` | Отрисовка курсора строительства |
| `setSelectedHex(hexId)` | Установка выделенного гекса |
| `setBuildingTypeToPlace(type)` | Установка типа постройки для размещения |

#### Процесс отрисовки:

1. Очистка canvas (`clearRect`)
2. Применение трансформации камеры (зум, смещение)
3. Для каждого гекса:
   - Конвертация axial → pixel координат
   - Отрисовка шестиугольника
   - Отрисовка постройки (если есть)
   - Отрисовка пользователей (если есть)
4. Отрисовка курсора строительства (если активен)

#### Отрисовка гекса:

```javascript
// 6 вершин шестиугольника (flat-top)
for (let i = 0; i < 6; i++) {
    const angle = (i * 60) * (Math.PI / 180);
    const x = hex.x + size * Math.cos(angle);
    const y = hex.y + size * Math.sin(angle);
    // ... добавление вершин в путь
}
```

### 4. `camera.js` — Управление камерой

**[ЧТО]** Управление панорамированием и зумом карты  
**[ЗАЧЕМ]** Навигация по большой карте 200×200 гексов со случайной генерацией

#### Основные компоненты:

```javascript
class CameraManager {
    constructor(canvas, hexGrid) {
        this.canvas              // HTML5 canvas элемент
        this.hexGrid             // Ссылка на HexGrid
        this.offsetX             // Смещение камеры по X
        this.offsetY             // Смещение камеры по Y
        this.zoom                // Текущий уровень зума (1 = базовый)
        this.minZoom             // Минимальный зум (0.3)
        this.maxZoom             // Максимальный зум (3)
        this.isDragging          // Флаг перетаскивания
        this.mode                // Режим: 'pan' или 'interact'
    }
}
```

#### Ключевые методы:

| Метод | Описание |
|-------|----------|
| `constructor(canvas, hexGrid)` | Инициализация камеры |
| `setupEventListeners()` | Настройка обработчиков мыши |
| `handleMouseDown(e)` | Начало перетаскивания (ЛКМ) |
| `handleMouseUp()` | Завершение перетаскивания |
| `handleMouseMove(e)` | Перемещение камеры |
| `handleWheel(e)` | Зумирование колесом мыши |
| `applyTransform(ctx)` | Применение трансформации к контексту |

#### Управление:
- **ЛКМ (drag)** — панорамирование карты
- **Колесо мыши** — зум (0.3× до 3×)
- **Режимы:** `pan` (навигация), `interact` (взаимодействие с объектами)

#### Ключевые методы:

### 6. `userManager.js` — Управление пользователями

**[ЧТО]** Создаёт, хранит и обновляет пользователей на гексах  
**[ЗАЧЕМ]** Симуляция активности на карте, пользователи могут перемещаться

#### Основные компоненты:

```javascript
class UserManager {
    constructor(hexGrid) {
        this.hexGrid             // Ссылка на HexGrid
        this.users               // Массив всех пользователей
        this.maxUsers            // Максимум (1.5 × число гексов)
        this.maxUsersPerHex      // Максимум на гекс (6)
        this.nextUserId          // Счётчик для уникальных ID
    }
}
```

#### Структура данных пользователя:

```javascript
{
    id: number,           // Уникальный ID
    hexId: number,        // ID гекса где находится
    createdAt: number,    // Время создания
    color: string,        // Цвет для отрисовки
    moveCooldown: number  // Задержка перед следующим перемещением
}
```

#### Ключевые методы:

| Метод | Описание |
|-------|----------|
| `constructor(hexGrid)` | Инициализация менеджера |
| `createUser()` | Создание нового пользователя |
| `getAvailableHexes()` | Поиск гексов с местом (< 6 пользователей) |
| `populateInitialUsers(count)` | Заполнение карты начальными пользователями |
| `updateUsers()` | Обновление состояния (перемещение) |
| `generateRandomColor()` | Генерация случайного цвета |

#### Лимиты:
- **Максимум пользователей:** 1.5 × количество гексов
- **На один гекс:** до 6 пользователей
- **Обновление:** каждые 1 секунду (перемещение)

## 🎮 Игровой цикл

```javascript
// В game.js
gameLoop() {
    this.gridRenderer.render();  // Отрисовка
    requestAnimationFrame(() => this.gameLoop());  // Следующий кадр
}
```

**Частота обновления:** ~60 FPS (зависит от монитора)

## 📝 Комментарии в коде

Все модули используют единую систему комментариев:

- **[ЧТО]** — Что делает код
- **[ЗАЧЕМ]** — Зачем это нужно
- **[PLAN]** — Планы на улучшение
- **[ERROR]** — Исправленные ошибки

Пример:
```javascript
// [ЧТО] Создаём центральный гекс с координатами (0, 0)
// [ЗАЧЕМ] Это точка отсчёта для всей сетки
// [PLAN] Добавить случайный тип местности
const centerHex = this.createHex(0, 0, hexId++);
```
