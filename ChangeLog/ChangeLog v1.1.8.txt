WeaveNet ChangeLog v1.1.8
=========================
Дата: 2024-01-XX

Выполненные задачи:
-------------------

[+] 1. Исправлена проблема видимости при множественных вышках/роутерах
    - Проблема: Покрытие загораживает гексы при установке нескольких вышек или роутеров
    - Решение:
      * Добавлен параметр coverageMode в метод drawCoverage() для выбора режима отрисовки
      * Режим 'main': градиентное заполнение с повышенной прозрачностью (5-18%)
      * Режим 'load': только контурная обводка без заполнения (пунктирная линия)
      * Основной метод draw() теперь использует режим 'load' для отрисовки зон покрытия
    - Файл: /workspace/WeaveNet/src/game.js

[+] 2. Обновлена логика отрисовки зон покрытия
    - Изменения в drawCoverage():
      * Уменьшена прозрачность градиента в режиме 'main' (было 50%/25%/12%, стало 18%/9%/5%)
      * Добавлен режим 'load' с пунктирной обводкой (50% непрозрачности)
      * Толщина линии уменьшена с 2 до 1 пикселя в режиме 'main'
    - Файл: /workspace/WeaveNet/src/game.js

[+] 3. Оптимизирован порядок отрисовки
    - Новый порядок слоёв: гексы -> зоны покрытия (контуром) -> объекты
    - Такой порядок не перекрывает гексы полупрозрачными зонами
    - Игрок может видеть территорию под зонами покрытия
    - Файл: /workspace/WeaveNet/src/game.js

Технические детали:
------------------
- Метод drawCoverage() теперь принимает второй параметр coverageMode ('main' или 'load')
- По умолчанию используется режим 'main' для обратной совместимости
- В методе draw() явно указан режим 'load' для лучшей видимости карты
- Пунктирная линия в режиме 'load': setLineDash([5, 5])
- Цвета зон покрытия сохранены: синий (#4a9eff) для роутеров, оранжевый (#f59e0b) для вышек
